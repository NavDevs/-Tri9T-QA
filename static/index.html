Software

Tri9T QA Engine is an AI-powered pipeline designed to automate Quality Assurance for strict regulatory environments. It parses complex requirement specifications, vectorizes them for semantic search, and dynamically generates comprehensive, audit-ready test cases using Large Language Models.

Technical Architecture

1. Dynamic PDF Ingestion

Built for real-world FDA/ISO workflows, this engine processes raw, unstructured PDFs rather than plain Markdown. It dynamically reconstructs the engineering requirement hierarchy by running heuristics on bytecode font sizing and layout.

2. RAG & Context Locking

Leverages a hybrid SQL (SQLite) and NoSQL (TinyDB) approach. It builds a versioned structural tree, allowing users to run targeted searches to isolate specific requirement nodes and lock them in as a strict context window for the LLM.

3. Staleness Auditing

Implements robust version control. If a new version of a product manual is uploaded, the system computes SHA-256 hashes of the tree nodes to instantly flag older test cases as [ STALE ] if their source requirement was altered.
